# 商家运营控制台设计

## 背景

当前系统已经具备 NFC 落地页、商家后台、管理员后台、平台跳转链接、评价池、评价生成任务、外部评论采集和发布点击统计等能力。

这次设计的目标不是继续做复杂工单流，而是把管理员侧升级为一个面向商家运营的控制台：先判断商家是否激活，再判断 NFC 写入的服务器链接是否可用、商家官方链接是否配置完整，再查看使用和转化数据，最后给出可执行的运营优化建议。

## 设计结论

控制台采用多页模式：

1. 总览页：看平台整体商家状态和关键风险。
2. 商家管理页：看商家列表、激活状态、官方链接配置、NFC 链接可用性和核心数据。
3. 商家详情页：看单个商家的基础资料、服务器落地链接、官方链接、NFC 交付信息和近期表现。
4. 使用明细页：看商家的访问、换一换、复制、发布点击和设备分布。
5. 转化分析页：看从 NFC 访问到发布点击的漏斗。
6. 运营建议页：根据数据规则给出优化建议。
7. 配置资料页：集中管理服务器落地链接、商家官方链接、NFC 写入状态、关键词、图片和评价生成偏好。

## 关键产品规则

### 三类链接必须分离

控制台里必须把三件事拆开：

| 类型 | 写在哪里 | 作用 | 是否由商家提供 |
| --- | --- | --- | --- |
| 服务器落地链接 | 我方服务器生成，展示给管理员复制 | 顾客访问入口，承载落地页、行为日志、商家状态校验和后续跳转 | 否 |
| NFC 写入链接 | 写入 NFC，内容就是商家的服务器落地页路由 | 顾客碰 NFC 后先进入我方服务器，由服务器判断商家是否激活、路由是否可用 | 否 |
| 商家官方链接 | 配置在后台，作为平台跳转目标 | 顾客从落地页点击后进入商家的官方店铺、评价页或平台页 | 是 |

NFC 不能直接写商家的官方链接。NFC 写我方服务器链接，系统才能统计访问、判断商家是否激活、判断路由是否可访问，并在商家官方链接缺失时给出可控提示。

第一版不为 NFC 物料生成单独路由。每个商家或门店都有自己的服务器落地页路由，例如 `/landing/:store_uuid`。NFC 写入这个服务器落地页路由；如果商家未激活，对应路由直接返回错误页或不可用状态，不继续展示评价和官方跳转入口。

### 激活状态、官方链接配置、NFC 可用性必须分离

`激活状态` 是商家的业务状态，只表示该商家是否处于可运营、可使用、可统计的状态。

`官方链接配置状态` 是配置完整度，只表示是否已经配置商家的官方跳转链接。

`NFC 可用性` 是交付物状态，只表示顾客碰 NFC 后是否能打开对应商家的服务器落地页路由。第一版不识别具体 NFC 物料，只判断该商家的 NFC 写入链接和落地页路由是否可用。

三者没有推导关系：

| 场景 | 激活状态 | 官方链接配置 | NFC 可用性 | 产品含义 |
| --- | --- | --- | --- | --- |
| 新建后准备运营 | 已激活 | 未配置 | 可用 | NFC 能打开商家服务器落地页，但还不能跳到商家官方页 |
| 提前配置资料 | 未激活 | 已配置 | 未写入或未启用 | 资料已准备好，但商家暂不启用 |
| 正常运营 | 已激活 | 已配置 | 可用 | 可正常接待 NFC 访问、发放评价并跳转官方链接 |
| NFC 路由异常 | 已激活 | 已配置 | 不可用 | 商家资料没问题，但 NFC 写入链接或服务器落地页不可用 |
| 停止合作 | 未激活 | 已配置或未配置 | 不可用 | 商家停用，服务器落地页路由返回不可用，链接配置只作为历史资料展示 |

列表和详情页必须同时展示这三个字段，避免运营误以为“配置了官方链接就等于激活”，或误以为“商家已激活就代表 NFC 路由可用”。

### 第一版状态枚举

商家状态只保留两个值：

| 状态 | 含义 | 推荐展示 |
| --- | --- | --- |
| 已激活 | 商家处于可运营状态 | 绿色状态点或标签 |
| 未激活 | 商家暂停、未上线或不可运营 | 灰色状态点或标签 |

不设计负责人、SLA、待处理、联调中、试运行等工单状态。

### 官方链接配置状态枚举

| 状态 | 判定 | 推荐展示 |
| --- | --- | --- |
| 已配置 | 至少有一个可用商家官方链接 | 蓝色或中性色标签 |
| 未配置 | 没有可用商家官方链接 | 橙色提示标签 |

如果实现层已有平台入口状态，第一版可以把“有官方链接但被禁用”归为“未配置可用链接”，并在详情页配置区展示具体原因。

### NFC 可用性枚举

| 状态 | 判定 | 推荐展示 |
| --- | --- | --- |
| 可用 | NFC 已写入商家服务器落地页路由，路由可访问，商家已激活 | 绿色标签 |
| 未写入 | NFC 还没有写入服务器链接 | 灰色标签 |
| 未绑定 | NFC 写入链接存在，但不是当前商家的服务器落地页路由 | 橙色标签 |
| 不可用 | 链接无法解析、商家未激活或落地页不可访问 | 红色标签 |

第一版展示商家维度的 NFC 状态：`已写入 / 未写入 / 不可用`。如果同一商家后续有多个 NFC 物料，后台可以记录数量，但路由判断只看商家的服务器落地页，不区分具体物料。

│ 风险队列                                                       │

### 总览页 KPI

| 指标 | 口径 |
| --- | --- |
| 总商家 | 系统内商家总数 |
| 已激活商家 | 激活状态为已激活的商家数 |
| 未激活商家 | 激活状态为未激活的商家数 |
| 官方链接已配置商家 | 已配置可用商家官方链接的商家数 |
| 官方链接未配置商家 | 未配置可用商家官方链接的商家数 |
| NFC 路由可用商家 | NFC 已写入服务器落地页，且路由可访问的商家数 |
| NFC 路由异常商家 | NFC 未写入、链接不匹配、商家未激活或服务器路由不可访问的商家数 |
| 使用商家数 | 周期内有 NFC 访问的已激活商家数 |
| 商家使用率 | 周期内有 NFC 访问的已激活商家数 / 已激活商家数 |
| 平均发布点击转化率 | 周期内发布点击数 / NFC 访问数 |

### 列表字段

| 字段 | 说明 |
| --- | --- |
| 商家名称 | 商家或门店主名称 |
| 门店 | 第一版可显示 1，后续支持一商家多门店 |
| 激活状态 | 已激活 / 未激活，独立字段 |
| 官方链接 | 已配置 / 未配置，独立字段 |
| NFC 链接 | 商家服务器落地页是否已写入 NFC，路由是否可访问 |
| 服务器链接 | 默认不放表格主列，可在行展开或右侧摘要里复制 |
| NFC 访问 | 周期内落地页访问数 |
| 官方点击 | 周期内点击商家官方链接或平台发布入口次数 |
| 转化率 | 发布点击 / NFC 访问 |
| 最近使用 | 最近一次 NFC 访问、复制或发布点击时间 |
| 操作 | 查看、配置官方链接、查看 NFC、启用、停用 |

### 筛选设计

| 筛选 | 用途 |
| --- | --- |
| 已激活 | 看正在运营的商家 |
| 未激活 | 看暂停或未上线商家 |
| 官方链接已配置 | 看配置完整商家 |
| 官方链接未配置 | 看缺官方跳转目标的商家 |
| NFC 异常 | 看 NFC 未写入服务器链接、写入链接不匹配或服务器落地页不可用的商家 |
| 已激活但未配置官方链接 | 第一优先级配置风险队列 |
| 已激活但 NFC 异常 | 第一优先级交付风险队列 |
| 已激活但近 7 天无访问 | 需要线下引导或 NFC 摆放检查 |
| 高访问低转化 | 需要优化评价文案或跳转链路 |

### 使用明细事件

| 事件 | 业务含义 |
| --- | --- |
| NFC 访问 | 顾客通过 NFC 或服务器落地链接进入落地页 |
| 领取评价 | 系统向顾客展示一条评价 |
| 换一换 | 顾客不满意上一条评价 |
| 复制 | 顾客认可并复制评价 |
| 发布点击 | 顾客从我方落地页点击商家官方链接、平台店铺或发布入口 |

## 页面五：转化分析

转化分析页用漏斗定位掉点。

```text
NFC访问 1,284
  ↓ 领取评价 1,120    领取率 87.2%
  ↓ 复制评价 518      复制率 46.2%
  ↓ 发布点击 364      发布点击转化率 28.4%
```

### 转化指标

| 指标 | 公式 | 解读 |
| --- | --- | --- |
| 领取率 | 领取评价次数 / NFC 访问次数 | 落地页是否顺利让顾客看到评价 |
| 换一换率 | 换一换次数 / 领取评价次数 | 文案贴合度，越高越说明不满意 |
| 复制率 | 复制次数 / 领取评价次数 | 文案可用性 |
| 发布点击转化率 | 发布点击次数 / NFC 访问次数 | 最终引导发布能力 |
| 官方链接点击率 | 发布点击次数 / 复制次数 | 复制后是否继续去商家官方页或平台 |
| 使用商家率 | 有访问的已激活商家 / 已激活商家数 | 平台整体使用情况 |

### 诊断规则

| 数据表现 | 可能原因 |
| --- | --- |
| NFC 访问低 | NFC 未写入服务器链接、商家落地页路由不可用、摆放位置不明显、员工没有引导 |
| 领取率低 | 落地页加载慢、首屏不清楚、平台选择阻塞 |
| 换一换率高 | 评价文案不贴合门店、关键词或语气配置弱 |
| 复制率低 | 文案像模板、太夸张、缺真实细节 |
| 复制高但发布点击低 | 官方链接缺失、按钮不清楚或平台跳转体验差 |
| 发布点击高但真实评论低 | 需要看外部评论采集和 C 验证明细 |

## 页面六：运营建议

运营建议不是泛泛提示，而是基于规则输出下一步动作。

```text
┌──────────────────── 运营建议 ────────────────────┐
│ 高优先级：商家已激活但未配置官方链接                  │
│ 原因：近 7 天无法完成发布跳转                         │
│ 建议动作：[配置官方链接]                              │
│ 预期影响：恢复发布点击转化链路                         │
├──────────────────────────────────────────────────┤
│ 高优先级：NFC 路由不可用                              │
│ 原因：顾客碰卡后无法进入我方服务器落地页                 │
│ 建议动作：[检查服务器路由] [重新写入 NFC 链接]            │
├──────────────────────────────────────────────────┤
│ 中优先级：换一换率高于 35%                            │
│ 原因：顾客频繁更换评价，当前文案不够贴合                 │
│ 建议动作：[优化关键词] [生成新一批评价]                  │
└──────────────────────────────────────────────────┘
```

### 建议规则

| 触发条件 | 优先级 | 建议 |
| --- | --- | --- |
| 已激活且未配置官方链接 | 高 | 配置商家官方链接 |
| 已激活且 NFC 路由不可用 | 高 | 检查服务器落地页路由、NFC 写入链接和商家激活状态 |
| 已激活且近 7 天 NFC 访问为 0 | 高 | 检查 NFC 是否写入服务器链接、线下摆放、门店员工引导 |
| 换一换率高于 35% | 中 | 优化关键词、品牌语气和生成方向 |
| 复制率低于 25% | 中 | 增加真实参考评论，重新生成评价 |
| 复制率高但发布点击转化率低 | 高 | 检查官方链接、按钮文案和平台跳转可用性 |
| 可用评价低于 10 条 | 高 | 生成新一批评价 |
| 外部评论采集失败 | 中 | 管理员检查外部商家 ID 和采集服务 |
| 移动端占比高但转化低 | 中 | 优化移动端首屏按钮和平台选择流程 |

## 信息层级和导航流

这套控制台的 3 秒扫描优先级是：

1. 这个商家现在能不能运营：看激活状态。
2. 顾客碰 NFC 能不能进来：看 NFC 可用性和服务器落地页路由。
3. 顾客进来后能不能跳到商家官方页：看官方链接配置。
4. 用得好不好：看访问、复制、官方点击和转化率。
5. 下一步做什么：看运营建议。

```text
总览
  ↓ 发现风险队列
商家管理列表
  ↓ 选中商家
右侧摘要
  ↓ 查看详情
商家详情
  ├── 概览：先判断激活 / NFC / 官方链接 / 转化
  ├── 使用明细：查每天发生了什么
  ├── 转化分析：定位漏斗掉点
  ├── 运营建议：进入下一步动作
  └── 配置资料：修改服务器链接、官方链接、NFC、内容资产
```

列表页只能把最重要的状态放到主表格：`激活状态`、`官方链接`、`NFC 链接`、`NFC 访问`、`官方点击`、`转化率`。服务器落地链接默认放在行展开或右侧摘要里，避免表格过宽；详情页顶部再展示完整服务器落地链接和复制动作。

## 用户旅程

| 步骤 | 用户做什么 | 用户感受 | 计划如何支持 |
| --- | --- | --- | --- |
| 进入总览 | 想知道今天哪里出问题 | 需要快速判断优先级 | 风险队列先显示已激活但缺官方链接、NFC 异常、无访问 |
| 打开商家列表 | 搜索一个商家或筛选异常 | 不想理解复杂工单状态 | 列表只显示激活状态、官方链接、NFC 链接这三类状态 |
| 查看详情 | 判断这家商家是否能正常使用 | 需要一个明确结论 | 详情顶部把激活、官方链接、NFC 链接可用性并排展示 |
| 看使用明细 | 想知道顾客有没有真的用 | 需要证据，不想猜 | 展示访问、领取、换一换、复制、官方点击和设备分布 |
| 看转化分析 | 想知道掉点在哪里 | 需要能定位问题 | 漏斗把 NFC 访问到官方点击拆成清楚步骤 |
| 执行建议 | 想知道下一步做什么 | 不想读长说明 | 建议卡提供动作按钮：配置官方链接、检查 NFC、生成评价 |
| 回到列表 | 想继续处理下一家 | 需要不丢上下文 | 保留筛选、选中状态和最近处理结果 |

## 响应式与可访问性

### 桌面端

桌面端使用左侧导航 + 主列表 + 右侧摘要。主列表承载扫描，右侧摘要承载选中商家的关键动作。列表列宽需要固定优先级，长商家名和长 URL 必须截断并提供复制按钮。

### 平板端

平板端保留列表和摘要，但摘要可以下移为详情抽屉。筛选区从一行变成两行，避免压缩表格文字。

### 移动端

移动端不展示完整表格，改为商家卡片列表。每张卡片只显示：

1. 商家名称。
2. 激活状态。
3. 官方链接状态。
4. NFC 链接状态。
5. 本周期 NFC 访问和官方点击。

详情页使用顶部 sticky 状态条展示 `已激活 / 官方链接 / NFC 可用性`，Tabs 横向滚动。高频动作放在底部操作栏：复制服务器链接、配置官方链接、检查 NFC 链接。

### 可访问性

1. 状态不能只靠颜色表达，必须有文字：已激活、未激活、官方链接未配置、NFC 异常。
2. 所有复制、配置、启用、停用按钮必须可键盘聚焦。
3. 表格需要保留语义表头，移动端卡片需要用定义列表或明确标签。
4. 触控目标不小于 44px。
5. 正文对比度不低于 4.5:1。
6. URL 不只显示省略号，必须提供复制按钮和悬浮/展开查看完整链接的方式。

### 实现注意事项

当前系统同时存在商家账号状态和门店状态。第一版产品口径建议如下：

## 空状态和异常状态

| 场景 | 页面表现 |
| --- | --- |
| 未配置官方链接 | 列表显示“官方链接未配置”，详情顶部显示配置提醒 |
| NFC 未写入服务器链接 | 列表显示“NFC 未写入”，详情页给出复制服务器落地页动作 |
| NFC 路由不可用 | 列表显示“NFC 异常”，详情页展示商家未激活、路由不可访问或链接未写入等原因 |
| 未激活 | 数据卡可显示历史数据；顾客访问该商家的服务器落地页路由时返回不可用错误页，不进入评价流程 |

| 没有评价库存 | 高优先级提示“生成新一批评价” |
| 外部采集失败 | 商家侧只显示数据积累中，管理员侧显示失败原因 |
| 指标分母为 0 | 转化率显示 `-`，不显示 0% 误导运营 |

## 视觉与交互原则

1. 页面是运营工具，不做营销型大首屏。
2. 列表优先展示可扫描字段：激活状态、官方链接、NFC 链接、访问、点击、转化率、最近使用。
3. 状态标签不要过多，第一版只保留激活状态、官方链接配置状态和 NFC 可用性。
4. 运营建议要能点击到配置或生成动作，不能只是文字提示。
5. 指标旁边必须有口径说明，避免运营误读。
6. 未配置、低转化、库存不足等问题用提示色，但不要把整页做成报警面板。

## 验收标准

1. 商家列表能同时看到“激活状态”“官方链接配置状态”和“NFC 链接可用性”。
2. 激活状态不再由官方链接是否配置、NFC 是否可用推导。
3. 支持按已激活、未激活、官方链接已配置、官方链接未配置、NFC 异常筛选。
4. 商家详情页能看到使用明细、转化漏斗和运营建议。
5. 运营建议至少覆盖未配置官方链接、NFC 路由不可用、无访问、换一换率高、复制率低、发布点击转化低、评价库存不足。
6. 指标分母为 0 时显示 `-` 或“数据不足”，不展示误导性百分比。
7. 未激活商家的服务器落地页路由必须返回不可用错误页，不展示评价、不跳转商家官方链接。
8. 第一版不引入负责人、SLA 或复杂工单状态。

## 后续扩展

1. 支持一商家多门店后，商家列表展示商家汇总，详情中增加门店维度切换。
2. 运营建议增加“已处理/已忽略”状态，形成轻量闭环。
3. 接入开放平台后，在配置资料页增加 AppID、密钥、Webhook、权限范围和调用日志。
4. 把外部真实评论采集和 C 验证结果纳入更完整的转化归因。

3. 商家官方链接只作为最终跳转目标配置。

工程实现时需要按上述路由行为落地：顾客碰 NFC 后，如果商家未激活，应看到清楚的不可用提示，而不是空白页、普通 404 或继续进入评价流程。具体 HTTP 状态码可在工程设计阶段确定。
